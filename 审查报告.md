## 一、代码质量审查结果（python-reviewer）

- 建议：使用 `ET.SubElement()` API 程序化构建

**H2. XML 构建函数严重重复（违反 DRY）**
- 位置：`sqt_tool.py:44-68`
- 两个函数 90% 代码相同，合并为参数化函数

**H3. main() 过长（84 行）**
- 位置：`sqt_tool.py:146-229`
- 超过 50 行限制，拆分为 UI 构建函数

**H4. 全部函数缺少类型注解**
- 全局，添加完整类型注解

### MEDIUM 级别
| 编号 | 问题 | 建议 |

| M1 | UI 颜色/字体硬编码 | 提取为模块级常量 |

## 二、安全审查结果（security-reviewer）

### HIGH 级别
**S1. 缺少输入大小限制（DoS 风险）**
- 位置：`read_items()` 第 165-167 行
- 建议：限制最多 1000 行，每行最多 200 字符

### MEDIUM 级别
| 编号 | 问题 | 建议 |
|------|------|------|
| S2 | `ET.parse` 未显式禁用 XXE | 创建 XMLParser 清空 entity |
| S3 | 文件写入静默覆盖 | 覆盖前用 askyesno 确认 |
| S4 | XML 控制字符未过滤 | 添加 `re.sub` 过滤 |
| S5 | 硬编码用户绝对路径 | 用空字符串兜底 |

### LOW/INFO
| 编号 | 问题 | 建议 |
|------|------|------|
| S6 | 批处理文件硬编码路径 | 仅保留降级路径 |
| S7 | 转义约定依赖（INFO） | 后续重构时纳入 |

### ✅ 已确认安全
- XML 注入 → `escape()` 正确转义 ✅
- 路径遍历 → 使用 OS 原生对话框 ✅
- 命令注入 → 无 `subprocess`/`eval` ✅
- 硬编码密码/密钥 → 未发现 ✅
